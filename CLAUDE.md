## Project Overview

RimePulse is a macOS menu bar app that displays real-time typing statistics from the [Rime input method](https://rime.im). It reads JSON stats files generated by a Rime Lua plugin and shows today's character count, typing speed, peak speed, 7-day history, and cumulative totals.

- **Platform**: macOS 14 (Sonoma)+
- **Language**: Swift 5.10 / SwiftUI
- **Build system**: Swift Package Manager
- **Bundle ID**: `com.jiefeng.rimepulse`

## Build Commands

```bash
make build          # Debug build (swift build)
make run            # Debug build + run
make release        # Release build
make clean          # swift package clean
make install        # Release build ‚Üí /usr/local/bin/RimePulse
make bundle         # Release build ‚Üí ~/Applications/RimePulse.app
```

No tests or linter are currently configured.

## Architecture

**Entry point**: `RimeStatsApp` (`@main`) uses `MenuBarExtra` with `.window` style to render the menu bar popover. `AppDelegate` sets `.accessory` activation policy (no dock icon) and prevents automatic termination.

**Data flow**: `StatsReader` (@Observable, @MainActor) ‚Üí reads JSON files ‚Üí publishes `today: TypingStats?` and `history: [TypingStats]` ‚Üí SwiftUI views react via Observation.

**File watching strategy** in `StatsReader`:
1. Try `DispatchSource` on the today file (`O_EVTONLY`, events: write/rename/delete)
2. If the file doesn't exist yet, fall back to watching the directory for file creation
3. On delete/rename (Lua plugin may recreate the file), automatically restart watching with a short delay
4. Directory watching uses exponential backoff retry (up to 30s) if the directory can't be opened

**View hierarchy**:
- `StatusBarLabel` ‚Äî menu bar text: char count, CPM, peak CPM (monospaced, emoji-prefixed)
- `StatsPanel` ‚Äî popover container: today section + history + totals + launch-at-login toggle + quit
  - `TodayStatsView` ‚Äî big char count, CJK/ASCII breakdown, 4 metric cards (speed, peak, active time, commits), new words
  - `HistoryListView` / `HistoryRowView` ‚Äî last 7 days in compact rows
  - `TotalStatsView` ‚Äî aggregated chars, duration, commits across all visible records

**Data sources** (read from `~/Library/Rime` by default, configurable via `~/.config/rimestats/config.json`):
- `typing_stats_today.txt` / `typing_stats_today.json` ‚Äî single JSON object, today's live stats (`.txt` is primary, `.json` kept as legacy)
- `typing_stats.txt` / `typing_stats.jsonl` ‚Äî one JSON line per day, historical records
- Key fields: `chars` (CJK + ASCII chars), `chars_cjk`, `chars_ascii`, `current_cpm`, `peak_cpm`, `burst_cpm`, `active_minutes`

**Model**: `TypingStats` uses `snake_case` CodingKeys to map from JSON. All fields are required (no optionals). `id` is derived from `date`. Decoder accepts legacy `words_en` in place of `chars_ascii` so old JSONL records still parse.

**Speed algorithm** (`rime/tstats.lua`):
- Per-second ring buffer (60 buckets, one char-count per second) ‚Äî sliding window sum over 15s for `current_cpm`, 60s for `peak_cpm`
- Empty seconds are lazily zeroed when the next commit crosses into a new second ‚Äî stopping typing naturally decays `current_cpm`
- `burst_cpm` = per-commit instantaneous speed `n * 60000 / compose_time_ms`, kept as the day's max (bounded by `BURST_MIN_MS = 200`, `BURST_MAX_MS = 120000`)
- Character counting in `count_text`: CJK codepoints and visible ASCII (`0x21..0x7E`) both counted by character, not by word
